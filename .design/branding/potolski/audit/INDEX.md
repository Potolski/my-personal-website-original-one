# Audit Index

> Phase: audit | Brand: potolski | Generated: 2026-04-17

Entry point for the potolski brand audit. The audit assesses the shipped personal site (`app/page.js`, `app/projects/page.js`, `app/layout.js`) against the repositioning described in `BRIEF.md` — from services-pitch to first-person, job-seeking engineer portfolio.

## Documents in this audit

1. **[Brand Inventory](./brand-inventory.md)** — catalog of every current token, asset, and copy sample (color hexes, typography, icons, motion, voice).
2. **[Coherence Assessment](./coherence-assessment.md)** — how well current elements work together. Composite score: 2.7/5. Visual system is internally consistent but misaligned with the repositioned strategy.
3. **[Market Fit](./market-fit.md)** — positioning vs. reference portfolios (Brian Lovin, Rauno, Emil Kowalski, Paco Coursey). The site currently pattern-matches to "portfolio template" more than to any of the references.
4. **[Equity Analysis](./equity-analysis.md)** — what is genuinely worth preserving vs. what is just inertia. Genuine equity: dark mode, serif/sans pairing (contingent on upgrade), recruiter-scannable Experience structure, the Kmino softening copy. Everything else is inertia.
5. **[Evolution Map](./evolution-map.md)** — PRIMARY DELIVERABLE. Element-by-element PRESERVE/EVOLVE/REPLACE with rationale tied to personas. Consumed by downstream phases.

## Headline findings

- The site is a coherent services-era personal site partly edited toward a new strategy. The move (editorial-minimal blockchain portfolio) is right; execution is undercooked.
- Three elements are genuine equity: dark mode, serif/sans pairing (if upgraded beyond Georgia), and the recruiter-scannable Experience structure.
- Five elements actively hurt the 90-second credibility test: centered profile-photo hero, green-checkmark Skills cards, section-divider orange bars, decorative gradient blobs on `/projects`, and the "Founder @ Kmino" hero label.
- Largest missing element: a writing surface. Reference portfolios all lead with writing; this site hides Medium behind a single button.
- Orange (`#f97316`) is inertia, not equity — it has zero external recognition. The BRIEF's framing ("evolve — open") is correct. The color can stay if used with discipline, but there is no equity argument against replacing it.

## How downstream phases should consume this

- **Strategy phase** should read `market-fit.md` for positioning gaps and `equity-analysis.md` for the "preserve" list.
- **Voice phase** should read the "Voice samples" section of `brand-inventory.md` (cleanest voice lives in the Hero tagline and About Me paragraph; replace the Skills section and Experience bullets).
- **Visual identity phase** should treat `evolution-map.md` as the spec — every element has a preserve/evolve/replace call with rationale.
- **Design system phase** should extract tokens from `brand-inventory.md` once the visual phase lands the final palette + type decisions.

## Next phase
Strategy → voice → visual → system. Each downstream phase should reference `evolution-map.md` as the canonical list of what survives and what changes.
